# Activating virtual environment

In order to have access to <AppInline /> commands, you need to activate the
python virtual environment named `venv` created during the installation of
dependencies.

## Windows

1. Go to <AppDirectory version={VERSION} />
2. Click on `start.bat`
3. You can now use [{APP_NAME} modules](./modules.mdx)

## Ubuntu / macOS

1. Open a terminal
2. Type `cd` followed by a `<space>` and the path to the <AppDirectory
   version={VERSION} />
3. Enter command `. venv/bin/activate`
4. You can now use [{APP_NAME} modules](./modules.mdx)

:::tip

You can drag and drop the <AppDirectory version={VERSION} /> icon to the
terminal instead of typing the path.

:::

## Manual activation

For advanced users, you can activate the virtual environment manually by
running the following commands

```bash
# bash / sh
. venv/bin/activate

# fish
source venv/bin/activate.fish

# powershell
.\venv\Scripts\activate
```
