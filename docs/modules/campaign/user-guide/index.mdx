#### `audio_path` Important Note

:::danger

<Fragment>
  For <WindowsIcon /> Windows users, ensure that the `audio_path` does **not**
  contain spaces to avoid disrupting the audio server's functionality.
</Fragment>

If you've already generated processing data, rectify the `audio_path` within
your configuration file and **refresh the configuration** to implement the
changes.

This issue is known and a fix is currently under development.

Apologies for any inconvenience caused!
:::
